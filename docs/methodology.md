# 적응형·컨텍스트 인식 SPC/FDC 방법론

이 문서는 각 수치가 **무슨 뜻이고 어떻게 계산되는지**를 비전문가도 납득하도록
설명한다. 모든 식은 `lib/spc`, `lib/fdc`, `lib/bench`의 순수 함수로 구현돼 있고
단위 테스트로 검증된다.

## 0. 두 종류의 한계 — 가장 중요한 구분

| | 관리한계(Control Limit) | 규격(Spec, USL/LSL) |
|---|---|---|
| 출처 | 공정 데이터에서 계산 | 엔지니어/고객이 지정 |
| 성질 | 컨텍스트 따라 **변함**(정상) | **고정** |
| 의미 | "평소와 같은가?"(안정성) | "합격인가?"(품질) |
| 구현 | `limits.ts`, `charts/*` | `spec.ts` |

관리한계가 변하는 것은 결함이 아니라 적응이다. 합격 판정의 기준선(규격)은
따로 고정돼 있어 의미가 보존된다.

## 1. 웨이퍼 점수 (관리도의 한 점)
한 컨텍스트(Recipe×Stage×Step×웨이퍼타입) 안에서 웨이퍼 한 장이 그 스텝을
도는 동안의 신호를 **평균**낸 값(`getWaferPoints`의 `avg(signal)`). 부가로
std/min/max/표본수를 함께 본다.

## 2. σ(시그마)와 I-MR 관리한계
- **σ_within (단기 변동)** = MR_bar / d2, d2=1.128. MR_bar는 연속 웨이퍼 차이의 평균.
  → "이웃 웨이퍼끼리 평소 얼마나 다른가."
- **σ_total (전체 변동)** = 표본 표준편차. → "전체적으로 얼마나 퍼져 있나."
- UCL/CL/LCL = CL ± 3·σ_within. 이 범위 안이면 정상 변동.

## 3. Trust Score — 한계 신뢰도 (참신점)
`ratio = σ_total / σ_within`.
- ratio < 1.5 → **높음**: I-MR 한계 타당.
- 1.5~3 → **보통**: 약한 드리프트/자기상관 가능.
- ratio > 3 → **낮음(경고)**: 드리프트·이질 모집단으로 한계가 비현실적으로 좁음.

> 실측: `4%H2N2_BKM/PM2 Stage1/step5/Gas2_Monitor` → σ_within=0.30, σ_total=2.07,
> ratio≈6.9 → 89점 중 71점(80%) 거짓 이탈. 이 지표가 자동 경고한다.

## 4. 베이스라인 동결 (`baseline.ts`)
공정이 안정했던 최저변동 구간을 골든으로 잡아 한계를 1회 산출·**동결**.
이후 데이터를 고정 한계에 비춘다. 매 런 전체로 재계산하면 불량이 한계를
오염시켜(정적 3σ의 함정) 의미가 사라진다.

## 5. 적응형 차트군 (`charts/*`) + 자동 추천 (`recommend.ts`)
| 차트 | 식/원리 | 강점 |
|---|---|---|
| I-MR | 개별값 + 이동범위 | 개별 가시성, 큰 시프트·스파이크 즉시 |
| EWMA | z_t=λx_t+(1−λ)z_{t-1}, λ=0.2 | 작은 지속 시프트 빠른 탐지 |
| CUSUM | C±=max(0,C±±(x−CL)/σ−k), k=0.5,H=5 | 작은 시프트 누적 탐지 |
| 잔차 | AR(1) 예측오차에 I-MR | 자기상관/드리프트 제거 |

추천 규칙: 자기상관 유의(|ρ₁|>2/√n) → 잔차 / Trust 낮음 → 잔차+컨텍스트경고 /
변화점 존재 → CUSUM / 그 외 → EWMA. 근거 문자열을 함께 제공.

## 6. 공정능력 (`spec.ts`)
- Cp = (USL−LSL)/(6σ_within), Cpk = min(USL−μ, μ−LSL)/(3σ_within) — 단기.
- Pp/Ppk = 같은 식의 σ_total 버전 — 전체.
- 1.33↑ 우수, 1.0~1.33 보통, 1.0↓ 위험. Cpk≫Ppk면 장기 드리프트 존재.

## 7. 다변량 FDC (`fdc/*`)
- **PCA**: 120여 신호를 표준화 후 주성분으로 압축(Jacobi 고유분해). 90% 분산 보존.
- **Hotelling T²** = Σ score_i²/λ_i. "센서 상관까지 고려한 종합 이상도"(모델 내부).
  한계: (k(n²−1))/(n(n−k))·F_{k,n−k,α}.
- **SPE/Q** = Σ 잔차². "모델로 설명 안 되는 새 패턴"(상관붕괴). 한계: Jackson-Mudholkar.
- **결합 기여도**(`contribution.ts`): SPE 기여(잔차²) + T² 기여(z_j·Σ loading·score/λ).
  모델 내부/외부 이상 모두 원인 신호를 지목 → "어느 센서가 범인인지" 순위.

## 8. 트레이스 FDC (`golden-trace.ts`, `dtw.ts`)
- **골든 엔벨로프**: 정상 트레이스 시점별 평균±kσ. 새 트레이스 이탈 비율로 알람.
- **DTW**: 길이가 다른 트레이스를 시간 신축 정렬해 형태 거리 비교(Sakoe-Chiba 밴드).

## 9. 벤치마킹 (`bench/*`) — 객관 평가 (필수)
라벨 없는 데이터의 한계를 **합성 결함 주입**으로 해결: 정상 시계열에 알려진
시프트/드리프트/스파이크/분산증가를 넣고(정답 라벨 보유) 측정.
- **정밀도/재현율/F1**: 위반 점 분류 성능.
- **탐지지연(MTTD)**: 결함 시작 후 첫 알람까지 점 수.
- **FAR**: 정상 점 중 거짓 알람 비율.
- **ARL₀**: 정상인데 평균 몇 점마다 거짓 알람(↑좋음, 3σ 이론 ≈370).
- **ARL₁**: 시프트 시 평균 몇 점 만에 탐지(↓좋음).

`npm run benchmark` 결과 요약(시드 고정, 재현 가능):
- 정적 3σ: 결함이 한계를 오염 → FAR↑·재현율↓(현행 함정 실증).
- 동결 I-MR: FAR↓, 큰 시프트·스파이크 즉시 탐지.
- EWMA/CUSUM: 작은 시프트(+1σ)에서 재현율·지연 압도.
- 잔차: 드리프트에서 거짓알람 억제(자기상관 공정 대상).
